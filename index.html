1. تحليل جدوى المشاريع التجارية

  • تقارير بصيغ متعددة

  • تحديد المنطقة الأفضل

3. لوحة إدارة متقدمة
  • إدارة البيانات

  • إحصائيات شاملة
  • سجلات العمليات
4. تقارير متقدمة
  • تصدير Excel/CSV/PDF
  • رسوم بيانية
  • إحصائيات تفصيلية
المرحلة 1: إعداد قاعدة البيانات

1.1 إنشاء قاعدة البيانات والجداول

ما يقدمه:

قاعدة بيانات business_feasibilit_1 مع 13 جدول رئيسي، فهارس محسّنة للأداء، ودعم UTF-8 الكامل للغة العربية.

كيف يعمل:

  1. إنشاء قاعدة البيانات: CREATE DATABASE business_feasibilit_1 CHARACTER SET utf8mb4
  2. الجداول الرئيسية: region_population_summary (بيانات السكان حسب المناطق)، city_population_summary (بيانات المدن)، region_activity_license_estimates (تراخيص الأنشطة)، region_commercial_summary (التسجيلات التجارية)، predictions (التحليلات المنفذة)، admin_users (حسابات الإدارة)، admin_logs (سجلات العمليات)
  3. الفهارس: إضافة فهارس على الحقول المستخدمة في الاستعلامات لتحسين الأداء
  4. حساب Admin افتراضي: إنشاء حساب admin/admin123 للوصول الإداري
الموقع في الكود: scripts/setup_database_new.php

1.2 إعداد ملفات التكوين

ما يقدمه:

ملفات تكوين منظمة لإعدادات الاتصال بقاعدة البيانات وملفات مشتركة للهيدر والفوتر.

كيف يعمل:

  1. app/config/config.php: بيانات الاتصال (host, dbname, username, password)
  2. app/config/database.php: اتصال PDO مع معالجة الأخطاء
  3. includes/header.php: هيدر موحد مع قائمة تنقل
  4. includes/footer.php: فوتر موحد
الموقع في الكود: app/config/, includes/
المرحلة 2: معالجة البيانات

2.1 تنظيف البيانات الأولية (Python)

ما يقدمه:

تنظيف ملفات Excel/CSV الأولية، توحيد الأعمدة، تحويل الصيغ (Wide → Long)، وحفظ البيانات المنظفة في data/clean/.

كيف يعمل:

  1. قراءة البيانات: من data/raw/ (ملفات السكان، التراخيص، التسجيلات التجارية، ISIC)
  2. التنظيف: إزالة المسافات الزائدة، توحيد أسماء الأعمدة، تحويل الأنواع (نصي → رقمي)، حذف الصفوف الفارغة
  3. التحويل: Wide Format → Long Format للتراخيص، حساب المجاميع (Totals)
  4. الحفظ: حفظ في data/clean/ بتشفير UTF-8 with BOM
الموقع في الكود: scripts/clean_data.py

2.2 بناء البيانات المجمعة (Python)

ما يقدمه:

تجميع بيانات السكان حسب المناطق والمدن، حساب النسب المئوية، تقدير التراخيص حسب المناطق، ودمج البيانات في ملفات موحدة.

كيف يعمل:

  1. تجميع السكان: prepare_population() - تجميع حسب المنطقة والمدينة، حساب النسب (pct_saudi, pct_female)
  2. تجميع التسجيلات: prepare_commercial() - حساب عدد التسجيلات، رأس المال، والنسبة الوطنية
  3. تقدير التراخيص: prepare_license() - حساب region_share وتقدير التراخيص الإقليمية
  4. الدمج النهائي: consolidate() - دمج جميع المقاييس في ملف واحد
الموقع في الكود: scripts/build_master_dataset.py

2.3 استيراد البيانات إلى قاعدة البيانات

ما يقدمه:

استيراد ملفات CSV المنظفة إلى الجداول، التحقق من صحة البيانات، وإدراج البيانات الافتراضية.

كيف يعمل:

  1. قراءة الملفات: قراءة ملفات CSV من data/aggregated/
  2. الاستيراد: استخدام LOAD DATA INFILE أو INSERT statements
  3. التحقق: التحقق من صحة البيانات قبل الاستيراد، معالجة الأخطاء والتكرارات
  4. البيانات الافتراضية: إدراج حساب admin (admin/admin123) وعوامل التقييم الافتراضية
الموقع في الكود: import_aggregated_data_new.php
المرحلة 3: تطوير الواجهة العامة

3.1 الصفحة الرئيسية (تحليل الجدوى)

ما يقدمه:

نموذج تحليل الجدوى التفاعلي مع حساب المنافسين تلقائياً وإرسال البيانات للتحليل.

كيف يعمل:

  1. النموذج: قوائم منسدلة للأنشطة والمناطق، حقل الميزانية، حقل المنافسين (يُملأ تلقائياً)
  2. JavaScript: عند تغيير النشاط/المنطقة - استدعاء api/competitors.php عبر AJAX، عرض عدد المنافسين
  3. الإرسال: التحقق من الحقول المطلوبة، إرسال إلى api/analyze.php، إعادة التوجيه إلى results.php?id={prediction_id}
  4. التصميم: Bootstrap 5 RTL، تصميم متجاوب، تنبيهات ديناميكية
الموقع في الكود: public/index.php

3.2 الحاسبة المتقدمة

ما يقدمه:

خوارزمية حساب متقدمة (100 نقطة) مع عرض تفصيلي للعوامل وتوصيات مخصصة.

كيف يعمل:

  1. الخوارزمية (100 نقطة): المنافسة (25)، الميزانية (25)، حجم السوق (20)، نوع النشاط (15)، الخبرة (10)، العوامل الإقليمية (5)
  2. الحساب: calculateSuccessScore() - حساب النقاط لكل عامل، مقارنة الميزانية بمتوسط التكلفة
  3. عرض النتائج: دائرة ملونة (250px) حسب النتيجة، تفصيل العوامل مع النقاط، توصيات مخصصة
  4. حفظ في قاعدة البيانات: إرسال إلى api/analyze.php، حفظ في predictions
الموقع في الكود: public/calculate.php

3.3 صفحة النتائج

ما يقدمه:

عرض شامل للتحليل مع رسوم بيانية تفاعلية وخيارات التصدير المتعددة.

كيف يعمل:

  1. جلب البيانات: قراءة predictions حسب id، فك JSON من market_analysis، تقسيم recommendations
  2. عرض النتائج: دائرة كبيرة (250px) ملونة حسب النتيجة، بطاقة معلومات المشروع، بطاقة تحليل السوق، قائمة التوصيات
  3. الرسوم البيانية: Chart.js - Doughnut (توزيع عوامل الجدوى)، Chart.js - Bar (مقارنة المؤشرات)
  4. خيارات التصدير: طباعة PDF (window.print())، تحليل جديد، العودة للرئيسية
الموقع في الكود: public/results.php

3.4 مقارنة المناطق

ما يقدمه:

مقارنة شاملة بين منطقتين إداريتين لتحديد الموقع الأفضل للمشروع.

كيف يعمل:

  1. اختيار المناطق: قائمتان منسدلتان للمنطقتين، التحقق من اختيار منطقتين مختلفتين
  2. جلب البيانات: إرسال POST إلى api/compare.php، جلب بيانات كل منطقة من region_population_summary
  3. حساب الفائز: calculateScore() - (السكان÷1M×40) + (السعوديون÷1M×30) + (نسبة السعوديين×20) + (نسبة الإناث×10)
  4. عرض المقارنة: بطاقتان جنباً إلى جنب، مخطط أعمدة مقارن، تحليل تفصيلي لكل منطقة، توصيات ذكية
الموقع في الكود: public/compare.php

3.5 لوحة القيادة العامة

ما يقدمه:

لوحة قيادة شاملة تعرض إحصائيات السوق السعودي ورسوم بيانية تفاعلية.

كيف يعمل:

  1. الإحصائيات (6 بطاقات): عدد المناطق، المدن، الأنشطة، التحليلات المكتملة، إجمالي السكان، التسجيلات التجارية
  2. الرسوم البيانية: Doughnut (توزيع السكان - أكبر 5 مناطق)، Bar (التسجيلات التجارية - أكبر 5 مناطق)
  3. الجداول: أكبر المناطق من حيث السكان، أكبر المناطق من حيث التسجيلات، آخر التحليلات (أحدث 5)
الموقع في الكود: public/dashboard.php

3.6 التقارير المتقدمة

ما يقدمه:

تقارير شاملة مع خيارات تصدير متعددة (Excel, CSV, PDF) ورسوم بيانية تفاعلية.

كيف يعمل:

  1. التقارير: تقرير السكان (جميع المناطق)، تقرير السجلات التجارية، أكثر الأنشطة (أكبر 10)، آخر التحليلات (أحدث 20)
  2. التصدير: Excel (XLSX.js - أوراق متعددة)، CSV (UTF-8 with BOM)، PDF (window.print() مع CSS محسّن)
  3. الرسوم البيانية: Pie (توزيع السكان - جميع المناطق)، Bar (الأنشطة الأكثر شيوعاً)
الموقع في الكود: public/reports.php
المرحلة 4: تطوير واجهات API

4.1 API التحليل

ما يقدمه:

واجهة برمجية لحساب نسبة الجدوى، تحليل السوق، توليد التوصيات، وحفظ النتائج في قاعدة البيانات.

كيف يعمل:

  1. استقبال البيانات: POST JSON (project_type, region, city, budget, competitors)
  2. حساب نسبة الجدوى: calculateFeasibilityScore() - حجم السوق (25%)، المنافسة (30%)، الميزانية (20%)، التطور الإقليمي (15%)، نوع النشاط (10%)
  3. تحليل السوق: getMarketAnalysis() - حجم السوق، عدد المنافسين، إمكانات السوق، مستوى المنافسة
  4. التوصيات: getRecommendations() - حسب النتيجة والمنافسة والميزانية
  5. حفظ في قاعدة البيانات: INSERT في predictions، إرجاع prediction_id
الموقع في الكود: public/api/analyze.php

4.2 API المنافسين

ما يقدمه:

حساب عدد المنافسين تلقائياً بناءً على نوع النشاط والمنطقة والمدينة.

كيف يعمل:

  1. استقبال البيانات: POST JSON (activity_type, region, city اختياري)
  2. البحث: إذا تم تحديد المدينة - SUM(license_count) FROM city_activity_license_estimates، إذا لم تُحدد - SUM(license_count) FROM region_activity_license_estimates
  3. الإرجاع: JSON {"competitors": 1523, "success": true}
الموقع في الكود: public/api/competitors.php

4.3 API المقارنة

ما يقدمه:

جلب بيانات منطقتين للمقارنة الشاملة.

كيف يعمل:

  1. استقبال البيانات: POST JSON (region1, region2)
  2. جلب البيانات: SELECT من region_population_summary لكل منطقة (جميع الحقول: السكان، النسب، إلخ)
  3. تنسيق البيانات: تحويل إلى أرقام (int/float)، إرجاع JSON منظم
الموقع في الكود: public/api/compare.php
المرحلة 5: تطوير لوحة الإدارة

5.1 صفحة تسجيل الدخول

ما يقدمه:

نظام تسجيل دخول آمن مع حماية من Brute Force وتسجيل جميع العمليات.

كيف يعمل:

  1. النموذج: حقل Username، حقل Password، زر تسجيل الدخول
  2. التحقق: SELECT من admin_users WHERE username=?، password_verify() للتحقق
  3. حماية Brute Force: تتبع المحاولات في $_SESSION['login_attempts']، بعد 3 محاولات - قفل لمدة 5 دقائق
  4. تسجيل العمليات: INSERT في admin_logs (action='LOGIN', username, ip_address, user_agent)
  5. إدارة الجلسات: $_SESSION['admin_logged_in'] = true، $_SESSION['admin_username']، $_SESSION['admin_id']
الموقع في الكود: admin/login.php

5.2 لوحة التحكم الإدارية

ما يقدمه:

لوحة تحكم شاملة تعرض إحصائيات النظام ورسوم بيانية ومعلومات النظام.

كيف يعمل:

  1. الإحصائيات (8 بطاقات): عدد المناطق، المدن، الأنشطة، التحليلات، إجمالي السكان، التسجيلات، متوسط الجدوى، تحليلات اليوم
  2. الرسوم البيانية: Bar (توزيع الأنشطة - أكبر 10)، Doughnut (توزيع المناطق - أكبر 10)
  3. جدول آخر التحليلات: SELECT أحدث 5 من predictions، عرض: نوع المشروع، المنطقة، المدينة، النسبة، التاريخ
  4. معلومات النظام: PHP Version، MySQL Version، استخدام الذاكرة، الوقت الحالي
الموقع في الكود: admin/dashboard.php

5.3 إدارة البيانات

ما يقدمه:

نظام CRUD كامل على جميع الجداول مع واجهة ديناميكية تتكيف مع بنية كل جدول.

كيف يعمل:

  1. اختيار الجدول: قائمة منسدلة (7 جداول متاحة)، إعادة تحميل الصفحة مع ?table=table_name
  2. عرض السجلات: DESCRIBE table_name لقراءة البنية، SELECT مع LIMIT 20 OFFSET (page-1)×20، Pagination
  3. إضافة سجل: ?action=add - عرض نموذج ديناميكي، إنشاء حقول حسب نوع البيانات (text → textarea، int → number، إلخ)، INSERT عند الإرسال
  4. تعديل سجل: ?action=edit&id=123 - جلب السجل، ملء النموذج بالبيانات الحالية، UPDATE عند الإرسال
  5. حذف سجل: تأكيد JavaScript، DELETE WHERE primary_key=?
الموقع في الكود: admin/data_manager.php
المرحلة 6: الأمان والحماية

6.1 أمان قاعدة البيانات

ما يقدمه:

حماية شاملة من SQL Injection ومعالجة آمنة لجميع المدخلات.

كيف يعمل:

  1. Prepared Statements: استخدام PDO مع prepare() و execute()، جميع الاستعلامات تستخدم placeholders (?)
  2. Input Sanitization: htmlspecialchars() للخروج، trim() للمدخلات، التحقق من الأنواع قبل الاستخدام
  3. Password Hashing: password_hash($password, PASSWORD_DEFAULT) عند الإنشاء، password_verify($password, $hash) عند التحقق، استخدام bcrypt (cost factor 10)
الموقع في الكود: جميع Controllers و Services

6.2 إدارة الجلسات

ما يقدمه:

نظام إدارة جلسات آمن يتحكم في مدة الجلسة وأمانها.

كيف يعمل:

  1. Session Configuration: session_start() في كل صفحة، session_regenerate_id() عند تسجيل الدخول
  2. Session Check: includes/session_check.php - التحقق من $_SESSION['admin_logged_in']، إعادة التوجيه إلى login.php عند عدم الدخول
  3. Session Timeout: التحقق من $_SESSION['login_time']، انتهاء الجلسة بعد 30 دقيقة من عدم النشاط
الموقع في الكود: includes/session_check.php, جميع صفحات Admin

6.3 حماية من الهجمات

ما يقدمه:

حماية متعددة الطبقات من Brute Force و XSS و CSRF.

كيف يعمل:

  1. Brute Force Protection: تتبع المحاولات في الجلسة، قفل بعد 3 محاولات لمدة 5 دقائق، رسالة واضحة للمستخدم
  2. XSS Prevention: htmlspecialchars() لجميع القيم المعروضة، عدم السماح بـ HTML في المدخلات
  3. CSRF Protection: (مستحسن إضافته) - توليد tokens فريدة لكل نموذج، التحقق من الـ token عند الإرسال
الموقع في الكود: admin/login.php, جميع Views

ما يقدمه:

تصميم متجاوب كامل يدعم جميع الأجهزة من الموبايل إلى الديسكتوب.

  1. Bootstrap 5 RTL: دعم العربية الكامل، Grid System متجاوب، Components جاهزة

كيف يعمل:

  2. التفاعلية: Tooltips عند التمرير، Legends قابلة للنقر، Responsive charts
  3. البيانات: جلب من PHP arrays، تنسيق JSON للـ Chart.js، تحديث ديناميكي
